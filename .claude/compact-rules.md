# Compact Rules

- Run /compact after every 3–4 completed tasks
- Before starting a new feature, always /compact
- Work on ONE component at a time — complete it fully before moving to next
- Use /clear when switching to a completely different feature

## Context Recovery (after /compact)
If context is cleared, read in this order:
1. `CLAUDE.md` — full project context and architecture
2. `.claude/coding-agent.md` — code rules and patterns
3. `lib/constants.ts` — all tools list
4. The specific file you need to change
